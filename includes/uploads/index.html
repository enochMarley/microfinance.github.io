Algorithms (Bubble Sort Algorithm)

An algorithm is a concise sequence of instructions to be performed within a finite amount of time and space. When an algorithm is invoked, it runs step by step, until all the instructions are executed.
Every algorithm has time complexity , which is the total amount of time it takes to finish execution, and space complexity, which is the total amount of memory space it requires to execute.Also, every algorithm has cases of complexity which are best case, average case and worst case.
The best case of an algorithm is the fastest time the algorithm takes to finish execution. This mostly happens when optimal inputs are chosen.For instance, the best case of a sorting algorithm will occur when the input for the algorithm is data that is already sorted. In this case the algorithm will execute fast because it wouldn't need to do any sorting of the input.
The average case of an algorithm is the arithmetic mean of the time of execution. Here the algorithm is run several times using different inputs and the running time for each input is calculated. The running times for all the inputs are added and the divided by the number of trials.
The worst case of an algorithm is the slowest time it takes for the algorithm to complete execution. For instance, the worst case of a sorting algorithm will occur when the input is reverse sorted data

Bubble Sort Algorithm

What Is The Bubble Sort Algorithm ?

The bubble sort is a sorting algorithm. The algorithm cycles and compares each pair of adjacent elements in a list and swaps them if they are not in order. The elements of the list gradually 'bubble' to their right position in the list.


How Does The Bubble Sort Algorithm Work?

Bubble Sort Algorithm cycles through a list and analyzes each pair of elements of that list from left to right, or from the beginning to the end of the list. It passes (iterates the list n times, where n is the number of elements in the list) and switches (iterates pairs of elements of the list in each pass). In each pair analyzed, if the element on the left side less than the element on the right side, the pair will remain the same and the next two pair will be analyzed. On the other hand, if the element on the left side greater than the element on the right side, their positions are switched, and the next two pair of elements are analyzed. The cycles iterates until a pass where no switch occurs.
As an example, consider the unsorted list of numbers A = [4, 11, 2, 8, 14]. This is how the Bubble Sort Algorithm will sort the numbers.

Pass 1:

  • Switch 1 : 4, 11, 2, 8, 14. No switching occurs because 4 is less than 11.
  • Switch 2 : 4, 2, 11, 8, 14. 2 and 11 are switched because 2 is less than 11.
  • Switch 3 : 4, 2, 8, 11, 14. 8 and 11 are switched because 8 is less than 11.
  • Switch 4 : 4, 2, 8 11, 14. No switching occurs because 11 is less than 14.
After pass 1 the updated list is A = [4, 2, 8, 11, 14]. Now we will proceed to pass 2.

Pass 2:
  • Switch 1 : 2, 4, 8, 11, 14. 2 and 4 are switched because 2 is less than 4.
  • Switch 2 : 2, 4, 8, 11, 14. No switching occurs because 4 is less than 8.
  • Switch 3 : 2, 4, 8, 11, 14. No switching occurs because 8 is less than 11.
  • Switch 4 : 2, 4, 8 11, 14. No switching occurs because 11 is less than 14.
After pass 2 the updated list is A = [2, 4, 8, 11, 14]. Now we will proceed to pass 3.

Pass 3:
  • Switch 1 : 2, 4, 8, 11, 14. No switching occurs because 2 is less than 4.
  • Switch 2 : 2, 4, 8, 11, 14. No switching occurs because 4 is less than 8.
  • Switch 3 : 2, 4, 8, 11, 14. No switching occurs because 8 is less than 11.
  • Switch 4 : 2, 4, 8 11, 14. No switching occurs because 11 is less than 14.
After pass 3 the updated list is A = [2, 4, 8, 11, 14]. Now we will proceed to pass 4.

Pass 4:
  • Switch 1 : 2, 4, 8, 11, 14. No switching occurs because 2 is less than 4.
  • Switch 2 : 2, 4, 8, 11, 14. No switching occurs because 4 is less than 8.
  • Switch 3 : 2, 4, 8, 11, 14. No switching occurs because 8 is less than 11.
  • Switch 4 : 2, 4, 8 11, 14. No switching occurs because 11 is less than 14.
After pass 4 the updated list is A = [2, 4, 8, 11, 14]. Now we will proceed to pass 5, which is the final pass.

Pass 5:
  • Switch 1 : 2, 4, 8, 11, 14. No switching occurs because 2 is less than 4.
  • Switch 2 : 2, 4, 8, 11, 14. No switching occurs because 4 is less than 8.
  • Switch 3 : 2, 4, 8, 11, 14. No switching occurs because 8 is less than 11.
  • Switch 4 : 2, 4, 8 11, 14. No switching occurs because 11 is less than 14.
After pass 5 the final list is A = [2, 4, 8, 11, 14].
It can be observed that after the 5 passes and 4 switches in each pass, the unsorted list A = [4, 11, 2, 8, 14] was sorted to A = [2, 4, 8, 11, 14] using the Bubble Sort algorithm. Close

Running Time And Efficiency

The time complexities of the algorithm are:
Best case : O(n). This is the case of already sorted list.
Average case : O(n 2).
Worst case : O(n 2). There will be n passes through the list, and each pass will have n-1 switches
n is the number of items.

The Bubble Sort algorithm is memory efficient because the all switches are done within the list itself. There is no new allocation of memory space. Also no new data structures are needed.

Merits And Demerits Of The Algorithm

Advantages of The Bubble Sort Algorithm

  • The Bubble Sort algorithm requires very little memory.
  • It is comprised of very few lines of codes.
  • It is good for testing whether a list is sorted or not since it has a best case running time of O(n)

Disadvantages of The Bubble Sort Algorithm

  • It is inefficient for large data set since it has average and worse case running times as O(n 2)
